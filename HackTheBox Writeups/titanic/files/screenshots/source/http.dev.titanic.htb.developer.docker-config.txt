<p dir="auto">Docker Compose is a powerful tool for defining and running multi-container Docker applications. It allows you to manage your Docker services with simple configuration files and commands. This repository provides an example of how to use Docker Compose to manage a Gitea instance, including the <code>docker-compose.yml</code> file used for configuration.</p>

<p dir="auto">Docker Compose uses YAML files to define the services, networks, and volumes for your application. This simplifies the configuration and makes it easy to understand and modify. Instead of managing individual <code>docker run</code> commands, you can define your entire stack in a single file.</p>
<h3 id="user-content-multi-container-applications" dir="auto"><a class="anchor" href="#multi-container-applications"><svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16" class="svg octicon-link" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .75.75 0 0 1 .018-1.042.75.75 0 0 1 1.042-.018 2 2 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.75.75 0 0 1-1.042-.018.75.75 0 0 1-.018-1.042m-4.69 9.64a2 2 0 0 0 2.83 0l1.25-1.25a.75.75 0 0 1 1.042.018.75.75 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .75.75 0 0 1-.018 1.042.75.75 0 0 1-1.042.018 2 2 0 0 0-2.83 0l-2.5 2.5a2 2 0 0 0 0 2.83"></path></svg></a>Multi-Container Applications</h3>
<p dir="auto">Modern applications often consist of multiple services that need to interact with each other. Docker Compose allows you to define and manage these services in a single file. You can start, stop, and scale services with simple commands, making it easier to manage complex applications.</p>
<h3 id="user-content-version-control" dir="auto"><a class="anchor" href="#version-control"><svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16" class="svg octicon-link" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .75.75 0 0 1 .018-1.042.75.75 0 0 1 1.042-.018 2 2 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.75.75 0 0 1-1.042-.018.75.75 0 0 1-.018-1.042m-4.69 9.64a2 2 0 0 0 2.83 0l1.25-1.25a.75.75 0 0 1 1.042.018.75.75 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .75.75 0 0 1-.018 1.042.75.75 0 0 1-1.042.018 2 2 0 0 0-2.83 0l-2.5 2.5a2 2 0 0 0 0 2.83"></path></svg></a>Version Control</h3>
<p dir="auto">Configuration files used by Docker Compose can be version controlled along with your application code. This ensures that your infrastructure is consistent across different environments (development, testing, production) and can be easily replicated or rolled back if needed.</p>

<p dir="auto">Docker Compose allows you to specify environment variables within your YAML configuration or in an external <code>.env</code> file. This makes it easy to manage different settings for different environments, such as development, testing, and production.</p>
<h3 id="user-content-volume-management" dir="auto"><a class="anchor" href="#volume-management"><svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 16 16" class="svg octicon-link" width="16" height="16" aria-hidden="true"><path d="m7.775 3.275 1.25-1.25a3.5 3.5 0 1 1 4.95 4.95l-2.5 2.5a3.5 3.5 0 0 1-4.95 0 .75.75 0 0 1 .018-1.042.75.75 0 0 1 1.042-.018 2 2 0 0 0 2.83 0l2.5-2.5a2.002 2.002 0 0 0-2.83-2.83l-1.25 1.25a.75.75 0 0 1-1.042-.018.75.75 0 0 1-.018-1.042m-4.69 9.64a2 2 0 0 0 2.83 0l1.25-1.25a.75.75 0 0 1 1.042.018.75.75 0 0 1 .018 1.042l-1.25 1.25a3.5 3.5 0 1 1-4.95-4.95l2.5-2.5a3.5 3.5 0 0 1 4.95 0 .75.75 0 0 1-.018 1.042.75.75 0 0 1-1.042.018 2 2 0 0 0-2.83 0l-2.5 2.5a2 2 0 0 0 0 2.83"></path></svg></a>Volume Management</h3>
<p dir="auto">Persistent storage is crucial for many applications. Docker Compose makes it easy to define and manage volumes, ensuring that your data is stored consistently and can be accessed by the appropriate services.</p>
